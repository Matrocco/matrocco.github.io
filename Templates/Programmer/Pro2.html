Mission

Un serveur offre des services de recherche de la présence de mots donnés par des clients dans des documents texte (txt), HTML, PDF et Excel (xslx). Le serveur affiche des informations supplémentaires utiles relatives à ces recherches, par exemple : ligne(s) où le mot a été trouvé, textes autour, etc.

Consigne

Commentaires et noms de variables seront écrits en anglais Un code clair, homogène, simple est attendu. Les programmes sont découpés en fonctions simples. La qualité de votre code est très importante pour l'évaluation. Lire cette synthèse PEP 8 – Style Guide for Python Code pour écrire votre code en la respectant. Ne mettez pas de valeurs constantes (par exemple 12) dans votre code, utilisez des constantes, c'est-à-dire des "variables" dont le nom est écrit en majuscule et dont vous n'allez pas changer les valeurs, par exemple MAX_NB_QUESTIONS = 12, cela permet également de voir à quoi correspondait ce 12). Attention à mettre des chemins relatifs, pas absolus. Exemple : PATH_XLS = "excel" Vous créerez un projet gitlab an ajoutant en tant que developer l'enseignant David Mercier sous le gitlab de l'université d'Artois. Mettez une docstring à vos fonctions. SAE 41

Mission

Réalisez en quadrinôme (groupe de 4) une maquette packettracer du réseau suivant correspondant au réseau d'une entreprise située sur deux sites distants :

Consigne

Trois réseaux séparés dans la partie LAN de chaque site : Service / Production / Admin. Chaque réseau Admin peut accéder à tous les autres réseaux y compris ceux de l'autre site.

Les réseaux Service et Production ne peuvent pas accéder aux autres réseaux y compris ceux de l'autre site.

Vous rédigerez un Write Up (WU) complet très précis des configurations (en mode texte pas de capture image).

Write up et maquette packettracer seront à déposer en fin de SAÉ. Il y aura une évaluation orale et écrite.

Le Tunnel sera un tunnel sécurisé IPSEC GRE (précisez tous les détails dans votre WU) Tous les membres du groupe doivent avoir travaillé sur ce réseau et ses équipements.

Ensuite chaque membre se spécialisera et étudiera 1 des 4 points suivants (à détailler dans votre WU) et mettra en place une maquette de démonstration :

  1. Sécurisation DNS avec DNSSEC
  2. Sécurisation WEB
  3. Tests de sécurité
  4. Recommandations ANSSI : Checklist des mesures retenues (fichier excel ci-dessous), exclues ou non applicables
